## TL;DR

- MedRAX is a **training-free ReAct agent** that wraps 7 specialist chest-X-ray tools (CheXagent + LLaVA-Med VQA, MedSAM + PSPNet segmentation, Maira-2 grounding, SwinV2+BERT report-gen, TorchXRayVision classification, RoentGen image-gen, plus DICOM utilities) around a GPT-4o planner. Tools are added by writing a description string -- no fine-tuning.
- Headline: **63.1% on the newly-introduced ChestAgentBench (2,500 six-choice MCQs over 675 Eurorad cases) vs 56.4% for GPT-4o, 57.9% for Llama-3.2-90B, 39.5% for CheXagent, 28.7% for LLaVA-Med** -- a uniform win across all 7 competencies (Detection 64.1, Diagnosis 62.5, Characterization 64.0).
- The honest read: ChestAgentBench is **GPT-4o-authored and GPT-4o-validated** while MedRAX uses GPT-4o as its backbone -- a circular setup. External wins on CheXbench Rad-Restruct (68.7 vs 53.9), SLAKE-CXR (**90.35% acc / 91.23% recall**), and MIMIC mF1-14 (**79.1**) are stronger evidence; but MIMIC macro-F1 (34.2) *loses* to M4CXR (40.0) -- MedRAX trails on rare findings.

## Motivation

Chest X-ray AI is fragmented. Classifiers, segmenters, grounders, and report generators each work in isolation, while general multimodal LLMs (GPT-4o, LLaVA-Med, CheXagent) unify the interface but hallucinate and skip systematic anatomical reasoning -- a safety problem at the scale of 4.2B exams read per year. The radiologist's actual workflow is multi-step: localise the abnormality, classify it, compare to priors, characterise, then diagnose. No single specialist or single VLM does all of this well. MedRAX's bet is that you can **keep the specialists as ground-truth tools and use an LLM as the radiologist-style planner**, getting hybrid behaviour without retraining anything.

## Core Innovation

- **ReAct loop over a specialist toolbox.** Each iteration is `Observe -> Reason -> (RequiresUserInput? / CanGenerateResponse?) -> SelectTools -> ExecuteParallel -> update memory`, exiting on answerable state, clarification need, or timeout. GPT-4o is the default planner; the architecture supports any tool-calling LLM.
- **Tool wrappers as the only prompt mechanism.** Each tool exposes `(name, natural-language description, input schema, execution logic)`. The description field is what teaches the LLM when to call the tool -- adding a new tool is a class definition, not a fine-tune.
- **Seven specialist categories** with strong off-the-shelf backbones: CheXagent (8.5M CheXinstruct samples), MedSAM (1.57M image-mask pairs), Maira-2 (MIMIC + PadChest + USMix), SwinV2+BERT (223K CheXpert Plus reports), TorchXRayVision DenseNet-121 (NIH + CheXpert + MIMIC + PadChest, 18 classes), RoentGen (Stable-Diffusion on MIMIC-CXR), and DICOM/visualization utilities.
- **ChestAgentBench**, a new benchmark of 2,500 six-choice MCQs over 675 expert-curated Eurorad cases, spanning 7 competencies (detection, classification, localisation, comparison, relationship, diagnosis, characterization) bundled into 5 question types. Auto-generated by GPT-4o conditioned on each case, then auto-validated by GPT-4o for format consistency, grounding in the clinical text, and verifiability. **No human re-review.**
- **Engineering.** Built on LangChain + LangGraph with parallel tool execution, memory caching across turns, quantized deployment, CPU/GPU distribution, and a Gradio UI that accepts DICOM. Evaluation parses A--F with regex and allows up to 3 retries.

**Honest read.** The tool-orchestration thesis is real, and the external-benchmark wins (CheXbench Rad-Restruct, SLAKE-CXR, MIMIC mF1-14) are the strongest evidence. The headline ChestAgentBench number is **partially circular**: GPT-4o both generates and validates the questions, and MedRAX uses GPT-4o as its backbone -- a slice of the 6.7-point gap over raw GPT-4o is almost certainly tools steering GPT-4o on questions GPT-4o itself authored. The MIMIC report-gen win is real for micro-F1 but *worse* on macro-F1, meaning MedRAX wins on common conditions and trails on rare ones -- a clinically important caveat the paper concedes only in passing. There is **no ablation table, no variance reporting, no statistical test, no latency/cost breakdown, no backbone-LLM ablation, and no radiologist user study**. Case studies are illustrative, not evaluative.

## Method & Architecture

![MedRAX architecture: GPT-4o ReAct loop with LangChain memory and 7 specialist CXR tools](/assets/images/paper/medrax/fig_p003_01.png)
*Figure 1: MedRAX architecture. A user query and optional CXR feeds a GPT-4o ReAct loop with LangChain memory and a toolbox of 7 specialist CXR models (Visual QA, Report Generation, Segmentation, Grounding, Classification, Image Generation, Utilities). Each tool is plugged in via a natural-language description -- no fine-tuning.*

**A. ReAct loop (Algorithm 1).**

Inputs: query $Q$, optional CXR images $I$, tool set $T$, memory $M$, timeout $t_{\max}$. Each iteration:

1. **Observe** the current state $(Q, I, M)$.
2. **Reason** about what is known, what is missing, and which tools could close the gap.
3. Check `RequiresUserInput?` -- if yes, ask for clarification and exit.
4. Check `CanGenerateResponse?` -- if yes, emit the final answer.
5. Otherwise **SelectTools** $\subseteq T$ and **ExecuteParallel** -- multiple specialists can run concurrently when they do not depend on each other.
6. Append outputs to $M$ and loop.

Exit conditions: answerable state, clarification need, or $t_{\max}$.

**B. Tool wrappers.**

Each tool is `(name, description, input_schema, execute)`. The description is the *primary* prompt mechanism -- it tells the LLM when to invoke the tool. Adding a new tool is just writing a wrapper class.

**C. Specialist toolbox.**

- **VQA:** CheXagent (Stanford CXR foundation model, trained on 8.5M CheXinstruct samples / 35 tasks) and LLaVA-Med (7B biomedical VLM).
- **Segmentation:** MedSAM (1,570,263 medical image-mask pairs across 10 modalities) and a PSPNet trained on ChestX-Det (3,578 NIH ChestX-14 images, 13 disease categories).
- **Grounding:** Maira-2 (7B VLM trained on MIMIC-CXR + PadChest + USMix) for phrase grounding.
- **Report Generation:** SwinV2 encoder + 2-layer BERT decoder trained on 223K CheXpert Plus reports (findings + impressions).
- **Classification:** TorchXRayVision DenseNet-121 (NIH ChestX-ray + CheXpert + MIMIC-CXR + PadChest), 18 pathology classes.
- **CXR generation:** RoentGen (Stable-Diffusion adapted on MIMIC-CXR).
- **Utilities:** DICOM I/O, plotting, visualization.

![Two-turn MedRAX interaction trace](/assets/images/paper/medrax/fig_p004_01.png)
*Figure 2: Example two-turn interaction. MedRAX first classifies the CXR then grounds a pleural-effusion finding; on the follow-up turn it segments the affected lung. The thought -> action -> tool -> response trace makes the agent's reasoning auditable -- the practical motivation for the tool-wrapper interface.*

**D. Orchestration.** Built on LangChain + LangGraph. Memory $M$ caches tool outputs across turns so the agent does not re-invoke the same classifier or segmenter unnecessarily. Supports quantized deployment, CPU/GPU distribution, and a Gradio UI that ingests DICOM.

**E. Evaluation harness.** Responses are parsed with regex to extract one of A--F; up to 3 retries on errors/timeouts; remaining invalid responses count as wrong.

## Experimental Results

![ChestAgentBench overview](/assets/images/paper/medrax/fig_p006_01.png)

Auto-generated and auto-validated by GPT-4o; no human re-review.*

**Table 1 -- ChestAgentBench accuracy (%) by competency.**

| Category | LLaVA-Med | CheXagent | Llama-3.2-90B | GPT-4o | **MedRAX** |
|---|---|---|---|---|---|
| Detection | 32.4 | 38.7 | 58.1 | 58.7 | **64.1** |
| Classification | 30.8 | 34.7 | 56.5 | 54.6 | **62.9** |
| Localization | 30.2 | 42.5 | 59.9 | 59.0 | **63.6** |
| Comparison | 30.6 | 38.5 | 57.5 | 55.5 | **61.8** |
| Relationship | 31.8 | 39.8 | 59.3 | 59.0 | **63.1** |
| Diagnosis | 29.3 | 33.5 | 55.9 | 52.6 | **62.5** |
| Characterization | 28.8 | 34.2 | 58.0 | 56.1 | **64.0** |
| **Overall** | 28.7 | 39.5 | 57.9 | 56.4 | **63.1** |

MedRAX wins every competency, with the largest absolute gains on Diagnosis (+9.9 over GPT-4o) and Classification (+8.3). But remember: GPT-4o is both the question author and the agent backbone. The +6.7 overall delta over raw GPT-4o conflates "tools genuinely help" with "tools steer GPT-4o on questions GPT-4o itself wrote."

**Table 2 -- CheXbench accuracy (%), external benchmark.**

| Subset | LLaVA-Med | CheXagent | Llama-3.2-90B | GPT-4o | **MedRAX** |
|---|---|---|---|---|---|
| VQA -- Rad-Restruct | 34.9 | 57.1 | 62.6 | 53.9 | **68.7** |
| VQA -- SLAKE | 55.5 | 78.1 | 74.0 | **85.4** | 82.9 |
| Fine-grained reasoning (OpenI) | 45.8 | **59.0** | 49.2 | 51.1 | 52.6 |
| **Overall** | 45.4 | 64.7 | 61.9 | 63.5 | **68.1** |

The CheXbench split is the cleanest part of the evaluation: external benchmark, no GPT-4o authorship contamination. MedRAX wins Rad-Restruct cleanly (+14.8 over GPT-4o) but **loses SLAKE to GPT-4o and loses fine-grained OpenI reasoning to CheXagent** -- the tool stack does not universally dominate.

**Qualitative case studies (Figure 4).**

*Figure 4: Two case-study comparisons against GPT-4o. Case 17576: GPT-4o picks "endotracheal tube" from central position alone; MedRAX combines classifier + report-generation + VQA outputs and answers "chest tube" despite LLaVA-Med dissenting with "nasogastric tube" -- the paper's evidence that the agent can override individual tool errors. The discussion concedes that "neither full tool reliance nor no tools yielded the best result" -- but the trade-off is never quantified.

## Limitations

**Authors admit:**

- Difficulty resolving contradictory tool outputs in fine-grained tasks.
- High computational overhead from running multiple specialist models in sequence or parallel.
- No robust uncertainty quantification.
- Open question of optimal tool-utilization balance.

**Additional gaps.**

- **ChestAgentBench is single-rater (GPT-4o) end-to-end.** No expert-radiologist sanity check on the 2,500 MCQs is reported, and the agent's GPT-4o backbone matches the question author -- the circularity risk inflates the headline number by an unknown amount.
- **No ablation table.** Per-tool contribution, backbone-LLM swap (Claude, Llama-3.2-90B, OSS LLM as planner), variance across seeds, confidence intervals -- all absent.
- **No latency / cost / token-count breakdown** vs end-to-end models, despite the clinical-deployment pitch. Multi-tool parallel execution is not free.
- **Macro-F1 regression on rare MIMIC findings** is a real safety concern; tail-pathology behaviour is under-explored.
- **Eurorad bias.** Cases are Europe-centric and published "interesting" findings -- not population-representative. ER (19.7%) and ICU (4.9%) samples are minorities despite being the highest-stakes clinical settings.
- **No radiologist user study** despite framing the system as a clinical co-pilot.

## Why It Matters for Medical AI

The contribution that survives the audit is **the engineering recipe**, not the headline benchmark number. MedRAX shows that a competent ReAct agent over off-the-shelf specialist CXR models can match or beat dedicated medical VLMs on multiple external benchmarks without any fine-tuning -- a deployment-friendly story when hospitals already have classifiers, segmenters, and report generators they trust. The released code, the natural-language tool-wrapper interface, and the ChestAgentBench dataset (the underlying *cases*, even if the MCQ layer is GPT-4o-authored) are real artifacts the field can build on. What does *not* hold up is the implied claim that tool orchestration generally beats specialist VLMs: SLAKE goes to raw GPT-4o, OpenI fine-grained reasoning goes to CheXagent, and macro-F1 on rare MIMIC findings goes to M4CXR. For clinical adoption the missing pieces are non-negotiable -- a non-GPT-4o-authored benchmark, per-tool ablations, a backbone-LLM swap, latency/cost numbers, and a radiologist agreement study.
